## 저는 이런 디자이너예요

제품 안에 숨어 있는 복잡한 문제를 빠르게 읽고, 팀이 함께 움직일 수 있는 구조로 정리하는 일을 좋아합니다.

사용자 인터뷰, 제품 데이터, 정보 구조, 프로토타입, 핸드오프, 배포 후 검증까지 하나의 제품 개선 과정으로 연결하려고 합니다.

- 모호한 문제를 구체적인 제품 방향으로 바꿉니다.
- 데이터를 근거로 삼되, 데이터가 만들어진 기준까지 확인합니다.
- AI 기능을 설계할 때 신뢰, 범위, 상태, 예외를 함께 봅니다.
- 복잡한 업무 과정을 단순하고 쓰기 쉬운 화면으로 번역합니다.
- 에너지 있게 소통하고, 필요한 맥락을 직접 파고듭니다.
- 숭실대학교 글로벌미디어학부를 전공했습니다.

## 제가 좋아하는 제품 문제

<table>
  <tr>
    <td width="33%" valign="top">
      <h3>문제 구조화</h3>
      <p>사용자 맥락과 비즈니스 목표를 함께 보고, 팀이 합의할 수 있는 문제 구조로 정리합니다.</p>
    </td>
    <td width="33%" valign="top">
      <h3>AI 경험 설계</h3>
      <p>AI답게 보이는 화면보다 사용자가 어디까지 믿어도 되는지, 어떤 상태를 봐야 하는지를 먼저 설계합니다.</p>
    </td>
    <td width="33%" valign="top">
      <h3>빠른 검증</h3>
      <p>상태와 예외가 많은 화면을 프로토타입으로 펼쳐 보고, 실제 사용 경로에서 검증합니다.</p>
    </td>
  </tr>
