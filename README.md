<table>

## 저는 이렇게 일합니다

- 인터뷰에서 들은 말을 그대로 믿기보다, 실제 사용 데이터와 다시 맞춰 봅니다.
- “사용자가 헷갈린다”는 말이 나오면 문구, 정보 구조, 다음 행동 버튼을 같이 봅니다.
- AI 기능을 만들 때는 결과물보다 신뢰 범위, 수정 가능성, 예외 상태를 먼저 정리합니다.
- 프로토타입은 예쁘게 보여주기 위해서가 아니라, 케이스를 빨리 펼쳐보기 위해 만듭니다.
- 모르는 도메인은 문서만 읽지 않고, 실제 사용자와 팀원이 쓰는 말을 따라가며 익힙니다.
